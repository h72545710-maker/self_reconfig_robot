# 简历项目排序建议

## 结论

这个自重构机器人项目可以作为秋招简历项目，而且建议放在项目经历的第一个或第二个。

如果你投递方向是：

- 嵌入式 Linux 软件开发
- C++ 软件开发
- 机器人软件开发
- 通用软件开发偏系统/工程方向

建议把它放在第一个。

如果你投递方向是：

- 单片机开发
- 硬件工程
- 传感器采集/检测设备

可以把阵列涡流项目放第一个，自重构机器人项目放第二个。

## 推荐项目顺序

### 方案 A：嵌入式 Linux / C++ / ROS2 方向

1. 自重构机器人多模块协同控制与 ROS2 桥接系统
2. 基于阵列涡流的金属缺陷实时成像系统
3. 小型管道涡流缺陷检测系统

这个顺序突出：

- C++ 工程能力
- Linux/树莓派部署
- ROS2
- 通信协议
- 控制算法
- 可视化和系统联调

### 方案 B：嵌入式硬件 / 单片机方向

1. 基于阵列涡流的金属缺陷实时成像系统
2. 自重构机器人多模块协同控制与 ROS2 桥接系统
3. 小型管道涡流缺陷检测系统

这个顺序突出：

- 硬件设计
- STM32/采集控制
- 上位机软件
- PCB 和工程调试

## 对研究生秋招是否够

作为“一个项目”是够的。

但作为研究生秋招简历，最好不要只靠这一个项目。你的简历里已经有两个偏硬件/检测系统项目，这个新项目刚好补上：

- Linux C++ 软件工程能力
- ROS2/机器人软件栈
- 控制算法和任务调度
- 树莓派部署和系统联调

这样组合起来，比单纯堆两个涡流检测项目更均衡。

## 项目定位建议

不要把它说成“我做了一个完整机器人”。

要把它说成：

> 我实现了一个面向自重构机器人的上位机协同控制原型，并完成了从 C++ 控制仿真到 ROS2 topic 发布的部署验证。

这个说法真实、稳，而且面试官追问时你能讲得住。

## 简历篇幅建议

如果一页简历，建议这个项目写 5 条 bullet。

如果两页简历，可以写 6-7 条，并额外加一句项目描述。

推荐保留关键词：

- C++17
- Linux
- ROS2 Jazzy
- 树莓派
- 多模块协同
- leader 选举
- A* 路径规划
- 故障恢复
- UDP / UART
- Web 可视化

## 和原有项目的互补关系

原来的两个涡流检测项目体现：

- 硬件设计
- 采集控制
- STM32/FreeRTOS
- Qt/C++ 上位机
- 传感器检测系统

新的自重构机器人项目体现：

- Linux C++ 工程
- ROS2
- 分布式状态同步
- 多模块任务调度
- 路径规划和协同控制
- 树莓派部署

这两个方向组合起来，会更像“嵌入式软件/机器人控制/系统开发”候选人，而不是只做单片机或只做硬件。
