# 简历项目描述：自重构机器人多模块协同控制系统

## 项目名称

基于嵌入式 Linux 的自重构机器人多模块协同控制系统

## 推荐时间

2026 年 7 月 - 至今

## 项目描述

面向自重构救援机器人多模块协同作业需求，基于树莓派嵌入式 Linux 环境设计并实现了一套多模块通信与任务调度控制原型。系统将单体机器人抽象为独立模块节点，通过自定义二进制协议完成模块注册、心跳检测、状态同步、leader 选举、任务分配与故障重分配，并提供 Web 可视化界面展示模块状态、任务进度和关键事件。

## 简历 Bullet 版本一：嵌入式软件岗

- 基于 C++17 在树莓派 Linux 环境实现 `master_node` 与多个 `module_node` 的 UDP 多进程通信框架，完成模块注册、周期心跳、状态同步和 leader/follower 角色管理。
- 设计自定义二进制通信协议，包含帧头、长度、消息类型、模块 ID、序列号、payload 和 CRC16-CCITT 校验，实现错误帧丢弃、消息序列追踪和协议扩展。
- 实现 ROLE 控制命令 ACK 确认与超时重传机制，通过故意丢弃 ACK 验证 master 端重发流程，提高关键控制命令下发可靠性。
- 实现模块故障恢复逻辑，支持主动 `fault` 上报和心跳超时 `offline` 检测；当 leader 异常离线时，系统可自动重新选举 leader。
- 实现 `EXPLORE` 任务分配、任务状态上报和任务重分配机制，在原 leader 离线后将任务迁移至新 leader，并继续上报任务进度。
- 搭建 Web 可视化面板，通过 `state.json` 实时展示模块位置、在线状态、任务归属、任务进度和事件日志，完成树莓派端部署验证。

## 简历 Bullet 版本二：通用 C++/系统软件岗

- 使用 C++17 设计多进程分布式控制原型，将通信协议、状态管理、角色调度、任务调度和可视化状态导出解耦，提升系统可维护性和扩展性。
- 实现跨平台 UDP socket 封装，支持 Windows 本地调试和树莓派 Linux 部署，完成多节点注册、心跳和状态同步。
- 自研二进制协议编解码模块，采用显式小端序列化和 CRC16-CCITT 校验，避免结构体内存布局差异导致的跨平台兼容问题。
- 实现可靠命令下发机制，master 维护待确认命令状态，支持 ACK 匹配、超时检测、重传次数限制和异常日志输出。
- 设计任务状态机，支持任务下发、ACK 确认、进度上报和 leader 故障后的任务重分配，验证多节点系统的容错调度能力。
- 编写 PowerShell/Bash 自动化脚本，支持本地演示、树莓派部署、故障演示、离线演示、ACK 重传演示和任务重分配演示。

## 技术栈

C++17、Linux、树莓派、UDP Socket、CMake/g++、PowerShell、Bash、HTML/CSS/JavaScript、CRC16、ACK/重传、心跳检测、任务调度、Web 可视化

## 简历中可压缩成一段

基于树莓派 Linux 和 C++17 实现自重构机器人多模块协同控制原型，设计 `master_node/module_node` 多进程架构和自定义二进制 UDP 协议，支持模块注册、心跳检测、CRC 校验、ACK/重传、leader 选举、故障离线检测、任务分配与任务重分配，并通过 Web 页面可视化模块状态、任务进度和关键事件，完成树莓派端部署验证。

## 面试可强调的亮点

- 不是单纯仿真，而是可部署到树莓派的嵌入式 Linux 控制框架。
- 自定义协议没有依赖 ROS2 或第三方通信框架，更能体现底层通信和系统设计能力。
- 同时处理了主动故障和静默掉线两种真实异常场景。
- ACK/重传、CRC、序列号让通信更接近真实设备通信。
- 任务重分配体现自重构机器人“容错协同”和“自恢复”的软件基础。
- Web 可视化解决了演示可读性问题。
